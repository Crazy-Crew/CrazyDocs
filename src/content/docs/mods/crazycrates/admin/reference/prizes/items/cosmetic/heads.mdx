<Tabs>
    <TabCard label="Default">
        <Aside type="tip">
            You can find the catalog of heads that can be used by visiting https://minecraft-heads.com/

            The identifier used in the "Default" example can be found by clicking any head.

            <Steps>
                1. Click any head on the website, for this example. We shall use https://minecraft-heads.com/custom-heads/head/127259-pumpkin

                2. Scroll to the bottom, look for `Minecraft URL`

                3. Copy the content after the url textures.minecraft.net/texture/**9cd583e846fd2196f50b3f95ffdc40ab99a09f5a4f2e840b8629d73edd979f2d**

                4. You now have the id: **9cd583e846fd2196f50b3f95ffdc40ab99a09f5a4f2e840b8629d73edd979f2d**
            </Steps>
        </Aside>

        ```yml
        Crate:
          Prizes:
            "1":
              # The material is required to be this for skulls to work.
              DisplayItem: "player_head"
              # The id that tells the plugin what skull to use.
              Player: "9cd583e846fd2196f50b3f95ffdc40ab99a09f5a4f2e840b8629d73edd979f2d"
              # Configure multiple items with 10 different skins if you wish.
              Items:
                "1":
                  # The material is required to be this for skulls to work.
                  material: "player_head"
                  # Extra options to configure the items in this pool.
                  settings:
                    # The id that tells the plugin what skull to use.
                    player: "9cd583e846fd2196f50b3f95ffdc40ab99a09f5a4f2e840b8629d73edd979f2d"
        ```
    </TabCard>

    <TabCard label="Player">
        <Aside type="caution">
            This uses a player name, and this player must exist on namemc.com.

            If the player is not on the server, a web request is made which can be heavy, but it's offloaded to another thread.

            In the future, I shall implement caching since there no need to keep querying web services.
        </Aside>

